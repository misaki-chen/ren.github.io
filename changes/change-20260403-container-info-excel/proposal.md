# Proposal: 柜号确认单信息录入 + Excel 导出工具

## Change ID
change-20260403-container-info-excel

## 背景与目标
物流操作中需要将柜号确认单信息整理成 Excel 文件。目前靠人工手动填写，效率低且易出错。需要一个工具支持：手动输入（主要方式）+ 图片 OCR 辅助识别 + 附件图片嵌入 + Excel 导出。

## 需求边界

### 要做的
- [x] 新建工具页面 `container_info.html`，加入 index.html 工具箱
- [x] **信息输入区**：12 个字段的表单，支持手动填写
- [x] **智能识别输入**：提供文本框，粘贴文本后自动识别字段并填充表单（参考 authorization_letter_multiple 的 keywordMap 模式）
- [x] **OCR 图片识别**：上传图片（柜号确认单等），OCR 提取文字后自动填充表单字段（仅用于识别信息，图片不进 Excel）
- [x] **附件图片上传**：上传柜门照、封条照等图片，这些图片会嵌入到 Excel 中
- [x] **预览功能**：按截图样式预览——左边表格（柜号确认单格式），右边附件图片横向排列
- [x] **导出 Excel**：单条记录导出，左边柜号确认单表格 + 右边嵌入附件图片
- [x] 识别结果可手动修改，未识别到的字段留空让用户填写

### 不做的 (YAGNI)
- 不做批量多条记录
- 不做后端服务（纯前端）
- 不做图片持久化存储
- 不做 PDF 导出

## 字段定义

### 主要字段（需识别/用户输入）

| 字段 | 别名关键词 | 数据特征 |
|------|-----------|---------|
| 订单号 | PO、单号、订单、订单号、订单编号 | PO:开头+数字 |
| 做柜时间 | DATE、日期、装柜时间、做柜日期 | 日期格式 MM/DD/HH:mm 等 |
| 工厂名称 | TO、收货方、工厂、客户 | 中文名称 |
| 订舱号 | 订仓号、仓号、booking、SO | 字母+数字混合 |
| 柜型 | 柜型号、箱型、size/type | 20GP/40HQ/40GP 等 |
| 柜号 | 箱号、container、柜号码 | 4字母+7数字 (如 SEKU6536421) |
| 封号 | 封条、seal、封条号 | 字母+数字 |
| 柜重 | 皮重、tare、自重 | 纯数字(kg) |
| 车牌 | 车牌号、车号、车牌号码 | 省份+字母+数字 |
| 司机 | 司机姓名、驾驶员、driver | 2-4字中文名 |
| 司机身份证号 | 身份证、证件号、ID | 18位数字(末位可X) |
| 司机手机号 | 电话、手机、联系方式、手机号码 | 1开头11位数字 |

### 默认值字段（有默认值，支持修改）

| 字段 | 默认值 | 说明 |
|------|--------|------|
| TO | 光普 | 收货方 |
| 装货地点 | 清溪 | 装货地点 |
| FROM | 伟航物流 | 发货方/物流公司 |
| 提柜地点 | SCT | 提柜码头 |
| 还柜地点 | SCT | 还柜码头 |

### 图片区（右侧，从 I 列开始横向排列）
- 附件图片按上传顺序从左到右排列
- 图片高度与表格区对齐

### 导出文件命名
- 格式：`{车牌}-司机资料.xls`
- 示例：`粤BES158-司机资料.xls`

### 空字段处理
- 没有信息的字段**留空**，不填占位符

## 技术方案
- OCR：Tesseract.js（前端离线 OCR，支持中文）
- Excel 生成：ExcelJS（支持图片嵌入、单元格合并、边框样式）
- 页面风格：与现有工具箱一致的紫色渐变主题

## 影响范围
- 新增页面: `container_info.html`
- 修改页面: `index.html`（添加工具卡片）

## 风险与注意事项
- Tesseract.js 中文识别准确率受图片质量影响，因此以手动输入为主、OCR 为辅
- Excel 嵌入图片需要使用 xlsx-js-style 或 ExcelJS 库（原生 SheetJS 社区版不支持图片嵌入），需评估库大小
