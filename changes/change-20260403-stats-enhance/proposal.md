# Proposal: Stats 统计面板增强 — 筛选 + 图表

## Change ID
change-20260403-stats-enhance

## 背景与目标
stats.html 已有基础的访客记录列表和概览卡片，但数据量增长后不便查看。需要增加筛选功能和可视化图表，提升数据分析效率。

## 需求边界

### 要做的
- [x] **筛选栏**：时间筛选（今天/7天/30天/全部）、页面筛选（下拉）、来源筛选（页面访问/按钮操作）、模糊搜索（IP/浏览器/系统）
- [x] **访问趋势折线图**：按天统计最近 30 天的访问量
- [x] **工具使用饼图**：各页面访问量占比
- [x] **操作热力柱状图**：各按钮点击次数排行
- [x] **设备分布环形图**：Desktop / Mobile / Tablet 占比
- [x] **浏览器分布饼图**：Chrome / Safari / Edge / 微信等占比
- [x] 筛选条件变化时表格和图表联动更新

### 不做的 (YAGNI)
- 不做用户登录/权限控制
- 不做实时刷新/WebSocket
- 不做数据导出功能

## 验收标准
- [ ] 筛选栏能正确过滤访客记录，表格实时更新
- [ ] 5 个图表正确渲染，数据与表格一致
- [ ] 筛选条件变化时图表联动刷新
- [ ] Chart.js 通过 CDN 引入，无额外依赖
- [ ] 页面加载完成后图表在 1 秒内渲染

## 技术方案
- 图表库：Chart.js (CDN)
- 数据源：复用现有 Google Sheets CSV 读取
- 筛选：纯前端 JS 过滤已加载的数据

## 影响范围
- 修改文件: `stats.html`（唯一改动）

## 风险与注意事项
- 数据量大时前端渲染可能卡顿，限制显示最近 500 条
